# Contributing to QuestForge

## Before opening a PR
- Open an issue first to discuss the change — this avoids wasted effort
- Keep PRs small and focused on one thing
- Test your change by opening QuestForge.html in a browser with Ollama running
- Don't change the visual theme without discussion

## What gets merged
- Bug fixes with a clear description of the problem
- New features that fit the DM tooling scope
- Model compatibility improvements
- Documentation improvements

## What won't get merged
- Changes that add external dependencies or require a build step
- Anything that breaks the single-file philosophy
- Style changes without prior discussion

## Response time
This is a side project maintained in spare time. PRs are reviewed when possible — usually within a few weeks.
